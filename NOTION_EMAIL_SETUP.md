# 📧 Notion 이메일 알림 설정 가이드

Azure AD 설정 없이 Notion의 자동화 기능으로 이메일 알림을 보내는 방법입니다.

## 🎯 개요

Notion의 자동화(Automation) 기능을 사용하여:
- ✅ 일정 등록 시 즉시 이메일 발송
- ✅ D-1 알림 자동 발송
- ✅ Azure AD 설정 불필요
- ✅ 무료로 사용 가능

## 📋 설정 단계

### 1. Notion 데이터베이스 속성 추가

기존 일정 데이터베이스에 다음 속성을 추가하세요:

1. **이메일 주소** (Email 타입)
   - 속성 이름: `이메일`
   - 타입: Email
   - 알림 받을 이메일 주소 입력

2. **알림 상태** (Select 타입)
   - 속성 이름: `알림상태`
   - 타입: Select
   - 옵션: `대기중`, `발송완료`, `발송실패`

3. **알림 날짜** (Date 타입)
   - 속성 이름: `알림날짜`
   - 타입: Date
   - D-1 날짜 자동 계산용

### 2. Notion 자동화 설정 - 즉시 알림

일정 등록 시 즉시 이메일을 보내는 자동화:

#### 2.1 자동화 생성

1. 데이터베이스 우측 상단 `•••` 클릭
2. `Automations` 선택
3. `+ New automation` 클릭

#### 2.2 트리거 설정

```
When: Page is added to database
Database: [일정 데이터베이스 선택]
```

#### 2.3 액션 설정

```
Do: Send email

To: [이메일] (속성 선택)
Subject: [일정 등록] {{제목}}
Body:
새로운 일정이 등록되었습니다.

📅 날짜: {{날짜}}
⏰ 시간: {{시간}}
📝 제목: {{제목}}
📄 설명: {{설명}}

Notion에서 확인하기: {{Page URL}}
```

#### 2.4 저장

- 자동화 이름: `일정 등록 알림`
- `Create automation` 클릭

### 3. Notion 자동화 설정 - D-1 알림

일정 하루 전에 알림을 보내는 자동화:

#### 3.1 자동화 생성

1. `+ New automation` 클릭

#### 3.2 트리거 설정

```
When: Property value changes
Database: [일정 데이터베이스 선택]
Property: 날짜
```

#### 3.3 조건 추가

```
If: 날짜 is 1 day from now
```

#### 3.4 액션 설정

```
Do: Send email

To: [이메일] (속성 선택)
Subject: [D-1 알림] {{제목}}
Body:
내일 일정이 있습니다! 🔔

📅 날짜: {{날짜}}
⏰ 시간: {{시간}}
📝 제목: {{제목}}
📄 설명: {{설명}}

준비하세요!

Notion에서 확인하기: {{Page URL}}
```

#### 3.5 저장

- 자동화 이름: `D-1 알림`
- `Create automation` 클릭

### 4. 프론트엔드 코드 수정

일정 등록 시 이메일 주소를 함께 저장하도록 수정:

#### 4.1 app.js 수정

기존 `NotionClient.createPage()` 메서드에 이메일 속성 추가:

```javascript
// 기존 코드에서 properties 부분 수정
properties: {
    '제목': {
        title: [{ text: { content: schedule.title } }]
    },
    '날짜': {
        date: { start: schedule.date }
    },
    '시간': {
        rich_text: [{ text: { content: schedule.time || '시간 미정' } }]
    },
    '설명': {
        rich_text: [{ text: { content: schedule.description } }]
    },
    // 이메일 속성 추가
    '이메일': {
        email: settings.recipientEmail || ''
    },
    // 알림 상태 추가
    '알림상태': {
        select: { name: '대기중' }
    }
}
```

## 🔧 고급 설정

### Formula로 D-1 날짜 자동 계산

`알림날짜` 속성을 Formula 타입으로 변경:

```
dateSubtract(prop("날짜"), 1, "days")
```

이렇게 하면 날짜가 입력되면 자동으로 D-1 날짜가 계산됩니다.

### 시간대별 알림

특정 시간에 알림을 보내려면:

1. Notion 자동화에서 시간 조건 추가
2. 예: "날짜가 내일이고 현재 시간이 오전 9시일 때"

## 📊 Notion 자동화 제한사항

### 무료 플랜
- 월 1,000회 자동화 실행
- 대부분의 개인 사용에 충분

### 유료 플랜
- 무제한 자동화 실행
- 더 복잡한 조건 설정 가능

## ✅ 장점

1. **간단한 설정**: Azure AD 불필요
2. **무료**: Notion 무료 플랜으로 가능
3. **안정적**: Notion이 직접 관리
4. **유연성**: Notion에서 직접 수정 가능

## ⚠️ 단점

1. **제한된 커스터마이징**: 이메일 템플릿 제한적
2. **실행 횟수 제한**: 무료 플랜은 월 1,000회
3. **지연 가능**: 즉시 발송이 아닐 수 있음

## 💡 추천 사용 시나리오

### Notion 자동화 추천
- 개인 사용
- 월 1,000회 이하
- 간단한 알림만 필요
- Azure AD 접근 불가

### Outlook MCP 추천
- 팀/회사 사용
- 대량 알림 필요
- 복잡한 이메일 템플릿
- Azure AD 접근 가능

## 🚀 다음 단계

1. Notion 데이터베이스에 속성 추가
2. 자동화 2개 생성 (즉시 알림, D-1 알림)
3. 프론트엔드 코드 수정 (이메일 속성 추가)
4. 테스트 일정 등록
5. 이메일 수신 확인

## 📝 테스트 방법

1. 채팅창에 입력:
   ```
   내일 오후 3시 테스트 회의
   ```

2. Notion 데이터베이스 확인:
   - 새 페이지 생성됨
   - 이메일 주소 입력됨
   - 알림상태: 대기중

3. 이메일 확인:
   - 즉시 알림 이메일 수신
   - 내일 오전 9시에 D-1 알림 수신 예정

## 🆘 문제 해결

### 이메일이 오지 않음

1. Notion 자동화 실행 로그 확인
2. 이메일 주소가 올바른지 확인
3. 스팸 폴더 확인
4. Notion 자동화 실행 횟수 확인

### 자동화가 실행되지 않음

1. 자동화가 활성화되어 있는지 확인
2. 트리거 조건이 맞는지 확인
3. Notion 플랜 제한 확인
