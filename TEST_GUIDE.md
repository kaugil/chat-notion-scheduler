# 🧪 테스트 가이드

chat-notion-scheduler의 모든 기능을 테스트하는 가이드입니다.

## 📋 테스트 체크리스트

### 1. 기본 설정 확인

- [ ] 기본 Notion Token 설정됨
- [ ] 기본 Database ID 설정됨
- [ ] 기본 이메일: ygkwon@kr.ibm.com
- [ ] Outlook 알림 활성화됨
- [ ] D-1 알림 활성화됨

### 2. Notion 데이터베이스 설정

#### 2.1 필수 속성 확인

Notion 데이터베이스에 다음 속성이 있는지 확인:

- [ ] **제목** (Title 타입)
- [ ] **날짜** (Date 타입)
- [ ] **시간** (Text 타입)
- [ ] **설명** (Text 타입)
- [ ] **이메일** (Email 타입) - 새로 추가
- [ ] **알림상태** (Select 타입) - 새로 추가
  - 옵션: 대기중, 발송완료, 발송실패

#### 2.2 Notion 자동화 설정

**자동화 1: 즉시 알림**
```
When: Page is added to database
Do: Send email
  To: [이메일]
  Subject: [일정 등록] {{제목}}
  Body: 
    새로운 일정이 등록되었습니다.
    
    📅 날짜: {{날짜}}
    ⏰ 시간: {{시간}}
    📝 제목: {{제목}}
    📄 설명: {{설명}}
```

**자동화 2: D-1 알림**
```
When: 날짜 is 1 day from now
Do: Send email
  To: [이메일]
  Subject: [D-1 알림] {{제목}}
  Body:
    내일 일정이 있습니다! 🔔
    
    📅 날짜: {{날짜}}
    ⏰ 시간: {{시간}}
    📝 제목: {{제목}}
```

### 3. 웹 애플리케이션 테스트

#### 3.1 로컬 서버 실행

```bash
cd chat-notion-scheduler
npm start
```

브라우저가 자동으로 열리고 `http://localhost:3000` 접속

#### 3.2 UI 확인

- [ ] 헤더 표시: "📅 채팅 일정 관리"
- [ ] 채팅 메시지 영역 표시
- [ ] 음성 입력 버튼 (🎤) 표시
- [ ] 텍스트 입력창 표시
- [ ] 전송 버튼 표시
- [ ] 설정 패널 (접혀있음)

#### 3.3 설정 패널 확인

설정 패널 클릭하여 펼치기:

- [ ] Notion 설정 섹션
  - Notion Token 입력됨
  - Database ID 입력됨
- [ ] Outlook 설정 섹션
  - "Outlook 이메일 발송 활성화" 체크됨
  - 이메일: ygkwon@kr.ibm.com
  - "D-1 알림 자동 발송" 체크됨

### 4. 텍스트 입력 테스트

#### 테스트 케이스 1: 기본 일정

**입력:**
```
내일 오후 3시 팀 회의
```

**예상 결과:**
- [ ] 사용자 메시지 표시
- [ ] "일정을 분석하고 있습니다..." 로딩 메시지
- [ ] 성공 메시지:
  ```
  ✅ 일정이 Notion에 등록되었습니다!
  
  📅 날짜: 2026년 7월 11일 (금)
  ⏰ 시간: 15:00
  📝 제목: 팀 회의
  
  📧 Notion 자동화를 통해 이메일이 발송됩니다!
  ⏰ D-1 알림이 Notion 자동화로 예약되었습니다!
  
  💡 Notion 데이터베이스에서 자동화 설정을 확인하세요.
  ```

**Notion 확인:**
- [ ] 새 페이지 생성됨
- [ ] 제목: "팀 회의"
- [ ] 날짜: 2026-07-11
- [ ] 시간: "15:00"
- [ ] 이메일: ygkwon@kr.ibm.com
- [ ] 알림상태: "대기중"

**이메일 확인:**
- [ ] ygkwon@kr.ibm.com으로 즉시 알림 이메일 수신
- [ ] 제목: "[일정 등록] 팀 회의"

#### 테스트 케이스 2: 복잡한 일정

**입력:**
```
2026년 7월 15일 오전 10시 30분 프로젝트 킥오프 미팅
```

**예상 결과:**
- [ ] 날짜: 2026년 7월 15일
- [ ] 시간: 10:30
- [ ] 제목: "프로젝트 킥오프 미팅"

#### 테스트 케이스 3: 상대적 날짜

**입력:**
```
모레 저녁 7시 저녁 약속
```

**예상 결과:**
- [ ] 날짜: 오늘 + 2일
- [ ] 시간: 19:00
- [ ] 제목: "저녁 약속"

#### 테스트 케이스 4: 요일 지정

**입력:**
```
다음주 월요일 오후 2시 정기 회의
```

**예상 결과:**
- [ ] 날짜: 다음 월요일
- [ ] 시간: 14:00
- [ ] 제목: "정기 회의"

### 5. 음성 입력 테스트

#### 5.1 브라우저 지원 확인

- [ ] Chrome 또는 Edge 브라우저 사용
- [ ] 마이크 연결 확인
- [ ] HTTPS 또는 localhost 환경

#### 5.2 음성 입력 테스트

**단계:**
1. 🎤 버튼 클릭
2. 마이크 권한 허용
3. "내일 오후 3시 회의" 말하기
4. 음성 인식 완료 대기

**예상 결과:**
- [ ] 버튼이 빨간색으로 변경 (listening 상태)
- [ ] "🎤 음성을 듣고 있습니다..." 메시지 표시
- [ ] 음성 인식 완료 후 텍스트 입력창에 표시
- [ ] 사용자 메시지로 표시: 🎤 "내일 오후 3시 회의"
- [ ] 버튼 원래 색상으로 복귀

**음성 입력 예시:**
- [ ] "내일 오후 3시 팀 회의"
- [ ] "모레 오전 10시 프로젝트 미팅"
- [ ] "금요일 저녁 7시 저녁 약속"

#### 5.3 오류 처리 테스트

**테스트 1: 음성 없음**
- [ ] 🎤 버튼 클릭 후 아무 말도 하지 않기
- [ ] "음성이 감지되지 않았습니다" 메시지 표시

**테스트 2: 마이크 권한 거부**
- [ ] 마이크 권한 거부
- [ ] "마이크 권한이 거부되었습니다" 메시지 표시
